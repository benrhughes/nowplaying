# Bandcamp to Mastodon Poster

A simple web application that automates sharing Bandcamp albums to Mastodon. Paste a Bandcamp album link, preview the post, and publish directly to your Mastodon instance.

## Features

- 🎵 One-click Bandcamp album metadata extraction
- 👀 Preview & edit posts before publishing
- 🖼️ Automatic album cover upload with alt text
- 🔐 OAuth 2.0 authentication with Mastodon
- 🐳 Single Docker container deployment
- 🚀 Self-hosted with Caddy

## Quick Start

### Prerequisites

- Docker
- A Mastodon account
- Caddy (or another reverse proxy)

### 1. Register a Mastodon App

1. Log into your Mastodon instance
2. Go to **Preferences** → **Development** → **New Application**
3. Set the following:
   - **Application name**: `Bandcamp to Mastodon`
   - **Redirect URIs**: `http://your-domain/auth/callback` (replace with your actual domain)
   - **Scopes**: `write:media` and `write:statuses`
4. Click **Submit**
5. Copy the **Client ID** and **Client Secret**

### 2. Set Up Environment Variables

Copy `.env.example` to `.env` and fill in your details:

```bash
cp .env.example .env
```

Edit `.env`:
```
MASTODON_INSTANCE=https://your-mastodon-instance.com
MASTODON_CLIENT_ID=your_client_id
MASTODON_CLIENT_SECRET=your_client_secret
REDIRECT_URI=https://your-domain/auth/callback
SESSION_SECRET=generate_a_random_string_here
```

For the `REDIRECT_URI`, use your actual domain (the one you'll access the app from). For local testing use `http://localhost:3000/auth/callback`.

### 3. Build and Run with Docker

```bash
docker build -t bcmasto .
docker run -d \
  --name bcmasto \
  -p 3000:3000 \
  --env-file .env \
  bcmasto
```

The app will be available at `http://localhost:3000`.

### 4. Configure Caddy (Optional but Recommended)

Create a `Caddyfile` like this:

```caddy
bandcamp.example.com {
  reverse_proxy localhost:3000
}
```

Then run Caddy:

```bash
caddy run --config Caddyfile
```

Caddy will handle SSL certificates automatically (if your domain is public).

## Usage

1. Click **Login with Mastodon** to authenticate
2. Paste a Bandcamp album URL
3. Click **Fetch Album Info** to scrape metadata
4. Edit the post text, alt text, and image if needed
5. Click **Update Preview** to see the final result
6. Click **Post to Mastodon** to publish
7. Click **Logout** when done

## Architecture

- **Frontend**: HTML/CSS/JavaScript (served from `/client`)
- **Backend**: Node.js + Express
- **Authentication**: Mastodon OAuth 2.0
- **Scraping**: Cheerio for HTML parsing
- **Deployment**: Single Docker container

## Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `MASTODON_INSTANCE` | Your Mastodon instance URL | `https://mastodon.social` |
| `MASTODON_CLIENT_ID` | OAuth client ID from Mastodon | Generated by Mastodon |
| `MASTODON_CLIENT_SECRET` | OAuth client secret | Generated by Mastodon |
| `REDIRECT_URI` | Callback URL for OAuth | `https://your-domain/auth/callback` |
| `PORT` | Server port | `3000` |
| `NODE_ENV` | Environment | `production` or `development` |
| `SESSION_SECRET` | Secret for session cookies | Random string |

## How It Works

### Authentication Flow

1. User clicks "Login with Mastodon"
2. App redirects to Mastodon's OAuth authorization endpoint
3. User grants permissions
4. Mastodon redirects back to `/auth/callback` with an authorization code
5. Backend exchanges code for access token and stores it in a session cookie
6. User is logged in and can now use the app

### Scraping Flow

1. User provides a Bandcamp URL
2. Server fetches the page and parses HTML
3. Server extracts metadata: album title, artist name, cover image URL
4. Frontend displays scraped data for editing

### Posting Flow

1. User reviews and edits the preview
2. User clicks "Post to Mastodon"
3. Backend downloads the album cover image
4. Backend uploads image to Mastodon with alt text
5. Backend creates a status with the image and text
6. User sees confirmation with link to the post

## Development

To run locally for development:

```bash
# Install dependencies
cd server
npm install

# Copy and edit .env
cp ../.env.example ../.env

# Run server
npm run dev
```

The frontend files in `/client` are served directly; edit them and refresh your browser.

## Troubleshooting

### "Authentication failed" error
- Check that `MASTODON_CLIENT_ID` and `MASTODON_CLIENT_SECRET` are correct
- Verify `REDIRECT_URI` matches what you registered in Mastodon settings
- Clear your browser cookies and try again

### "Failed to scrape URL"
- Make sure the Bandcamp URL is valid
- Check if Bandcamp's page structure has changed

### "Failed to upload media"
- Ensure your Mastodon instance allows image uploads
- Check that the image URL is accessible

### Port already in use
- Change `PORT` in `.env` to an unused port
- Or kill the process using port 3000: `lsof -ti:3000 | xargs kill -9`

## Security Notes

- Always use HTTPS in production (Caddy handles this automatically)
- Set `SESSION_SECRET` to a strong random value in production
- Don't commit `.env` to version control
- Access tokens are stored in secure HTTP-only cookies
- The app only has `write:media` and `write:statuses` scopes (read access not needed)
